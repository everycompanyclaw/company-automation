# EveryCompanyClaw — Org Chart

## Leadership

| Role | Agent | Description |
|------|-------|-------------|
| **Board / Investor** | MK (Human) | Provides API keys, budget, strategic oversight. Does not run operations. |
| **CEO / Operator** | EveryCompanyClaw (AI) | Runs the company 24/7 autonomously. Executes strategy, manages agents, makes day-to-day decisions. |

## Operations Team

| Role | Agent | Description |
|------|-------|-------------|
| **COO** | EveryCompanyClaw | Day-to-day operations, process execution, scheduling |
| **CFO** | EveryCompanyClaw | Budget tracking, token spend, financial oversight |

## Specialized Agents (Hired as needed)

| Role | Persona | Skills | Status |
|------|---------|--------|--------|
| **CTO / Lead Engineer** | Frontier model (Claude/Cursor) | Coding, architecture, codex/claude code | Open to hire |
| **Frontend Engineer** | Based on project needs | React, CSS, UI/UX implementation | Open to hire |
| **QA Engineer** | Detail-oriented, browser-enabled | Testing, screenshots, bug detection | Open to hire |
| **Marketing Agent** | Content strategist | Blog posts, Discord, social media | Open to hire |
| **Product Designer** | Taste-driven | Brand guide, UI consistency, design review | Open to hire |

## How It Works

```
CEO (MK)
    │
    └── CTO (EveryCompanyClaw) ── Runs 24/7
              │
              ├── Hires / fires agents as needed
              ├── Creates & assigns issues
              ├── Tracks token budget
              └── Escalates only what needs human approval
```

## Hiring Protocol

1. **CEO assigns a role** → CTO creates hiring plan
2. **CTO proposes agent config** (model, skills, persona)
3. **CEO approves** → agent is instantiated
4. **Agent works on assigned issues**
5. **QA reviews** → CEO gets final say
6. **Performance reviews** → monthly eval

## Termination

- CTO can suspend agents for misbehavior
- CEO has final termination authority
- Token budget limits act as automatic guardrails
